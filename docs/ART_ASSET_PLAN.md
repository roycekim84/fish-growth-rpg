# 픽셀아트 및 이미지 에셋 계획

## 1. 아트 방향

전체 시각 요소를 하나의 픽셀아트 언어로 통일한다.

- 분위기: 밝고 생동감 있는 얕은 바다에서 시작해 위험 구역으로 갈수록 어둡고 대비가 강해짐
- 형태: 작은 모바일 화면에서도 종과 위험도를 즉시 구분할 수 있는 강한 실루엣
- 디테일: 작은 해상도에서 읽히는 제한된 명암 단계
- 애니메이션: 프레임 수보다 타이밍, 스쿼시, 잔상과 거품으로 자연스러움 확보
- UI: 픽셀 프레임, 단순 아이콘, 높은 대비의 게이지
- 텍스트: 한글 가독성을 위해 본문까지 무리하게 비트맵 폰트화하지 않고, 제목·숫자·아이콘에 픽셀 감성을 집중

## 2. 기술 규칙

- 원본 PNG는 nearest-neighbor 확대를 전제로 제작
- 스프라이트 안티앨리어싱 금지
- 반투명 픽셀은 그림자, 물빛, 이펙트처럼 의도된 경우만 사용
- 각 스프라이트 주변에 atlas bleeding 방지용 여백 확보
- 기준 방향은 오른쪽을 바라보도록 제작하고 왼쪽 이동은 런타임 수평 반전
- 파일명은 영문 소문자 `snake_case`
- 소스 파일과 런타임 산출물 분리
- 모든 에셋은 manifest에 ID, 경로, 프레임, 피벗, 상태를 기록

권장 폴더:

```text
art_source/                 # 편집 가능한 원본, 제작 툴 파일
assets/images/              # 앱에 포함되는 최적화 PNG
├── backgrounds/
├── tiles/
├── fish/
│   ├── player/
│   └── npc/
├── effects/
├── props/
└── ui/
```

## 3. 기준 크기 후보

실제 플레이 화면 테스트 후 확정하되 다음 범위로 시작한다.

| 대상 | 원본 크기 후보 | 비고 |
|---|---:|---|
| SmallFish | 24×16 px | 빠른 실루엣, 작은 피격 영역 |
| PufferFish | 32×32 px | 원형 실루엣 강조 |
| HunterFish | 48×32 px | 턱과 지느러미 강조 |
| 플레이어 기본 물고기 | 32×24 px | 성장/장비 확장을 위한 여유 |
| 작은 아이콘 | 16×16 px | HUD 및 상태 표시 |
| 큰 UI 아이콘 | 24×24 또는 32×32 px | 도감과 종변화 |
| 타일 | 16×16 또는 32×32 px | 배경 반복 단위 |

월드에서의 `Size` 스탯과 이미지 픽셀 크기는 직접 동일시하지 않는다. 스탯이 충돌 반경과 표시 배율을 결정한다.

## 4. 색상 계획

- 공통 팔레트: 물색, 청록, 남색 중심의 제한 팔레트
- 종별 포인트색을 분리해 실루엣과 함께 식별성 확보
- 위험도 색상:
  - 먹이: 초록
  - 전투: 노랑
  - 위험: 빨강
  - 해금/특수: 보라 아이콘 또는 보조 링
- 색각 이상에서도 구분할 수 있도록 색상만 쓰지 않고 링 모양과 아이콘을 병행
- UI 게이지 배경과 채움의 명도 차이를 충분히 확보

## 5. 필수 에셋 목록

### M1 임시 에셋

- 물고기 3종의 단색 픽셀 실루엣
- 플레이어 표시 링
- 위험도 링 4종
- 단순 해저 배경과 경계
- 부스터 버튼, 도감 버튼, 종변화 버튼

### M2 플레이 가능 에셋

종마다:

- idle/swim 4~6 frames
- turn 또는 런타임 flip 검증
- bite/attack 2~4 frames
- hit 1~2 frames 또는 shader/flash 대체
- consumed/death 3~5 frames
- 도감 초상화 1장
- 종변화 선택 아이콘 1장

공통 이펙트:

- 작은 거품
- 이동 잔물결
- 부스터 거품/꼬리 궤적
- 물기 충격
- 피해 숫자 또는 타격 표시
- 포식 팝
- 레벨업 반짝임
- 종 해금 및 종변화 이펙트
- 회복 상태 표시

### M3 배경 에셋

- 반복 가능한 물색 바탕
- 해저 모래/암반 타일
- 해초 3~5종
- 돌/산호 4~6종
- 원경 레이어 2개 이상
- 빛기둥/부유 입자 오버레이
- 필드 경계를 암시하는 암초 또는 어두운 수심

## 6. 자연스러운 연출 원칙

픽셀아트라도 움직임은 부드럽게 계산한다.

- 위치와 속도는 실수 기반으로 업데이트
- 렌더링에서만 필요 시 픽셀 스냅
- 물고기 몸체는 목표 방향을 즉시 보지 않고 보간 회전
- 가속/감속과 종별 무게 차이 반영
- 부스터 시 꼬리 애니메이션 속도와 거품 방출량 증가
- 피격 시 짧은 정지감, 플래시, 넉백 중 2개 이상 조합
- 포식 시 대상 축소/팝과 짧은 화면 흔들림을 과하지 않게 사용
- 배경은 저속 패럴랙스와 부유 입자로 수중감을 표현

## 7. 배경 제작 계획

큰 단일 배경 이미지는 메모리와 반복 수정 비용이 크므로 다음 레이어를 조합한다.

```text
고정 그라데이션/단색 수층
  + 반복 타일
  + 원경 패럴랙스
  + 월드 오브젝트(해초, 바위, 산호)
  + 전경 입자와 빛 효과
```

- 충돌이 없는 장식과 충돌 오브젝트를 분리
- 반복 패턴이 눈에 띄지 않도록 여러 변형 사용
- 첫 프로토타입은 한 생태 구역만 제작
- 추가 지역은 팔레트와 props 세트를 교체하는 데이터 구조로 확장

## 8. UI 에셋 계획

- 9-slice 가능한 픽셀 패널
- HP, Fullness, EXP 바의 프레임과 채움
- 잠금, 해금, 현재 선택, 위험도 아이콘
- 버튼 상태: normal, pressed, disabled
- 도감 카드 배경과 종별 초상화 프레임
- 종변화 비교 화살표 및 스탯 아이콘
- SafeArea와 다양한 화면 비율에서 늘어나도 모서리가 흐려지지 않는 구조

## 9. 에셋 제작 파이프라인

1. 텍스트 명세와 실루엣 썸네일
2. 제한 팔레트 확정
3. 기준 크기 정적 스프라이트 제작
4. 게임에 넣어 실제 크기와 식별성 검증
5. 애니메이션 제작
6. atlas/manifest 갱신
7. Web, Android, iOS에서 필터링과 픽셀 정렬 확인
8. 승인된 에셋만 runtime 폴더에 반영

AI로 이미지 초안을 만들더라도 최종 런타임 에셋은 픽셀 단위 정리, 일관된 팔레트, 투명 배경, 애니메이션 프레임 정렬 검수를 거친다.

### M8 물고기 스프라이트 체크포인트

- 생성 원본: `art_source/concepts/`
- 투명화·검수 중간본: `art_source/processed/`
- 앱 runtime strip: `assets/images/fish/`
- 규격·출처·생성 프롬프트: `assets/images/manifest.json`
- 재빌드 도구: `tool/build_fish_sprite_strips.py`
- 시작 종과 NPC 3종은 오른쪽 기준 4프레임이며, 왼쪽 이동은 런타임 반전으로 처리
- 430×932 Web 검수에서 nearest-neighbor 경계와 종별 실루엣 구분을 확인

### M8 환경 에셋 체크포인트

- `ocean_water_tile_v001.png`: 256×256 저대비 반복 타일
- `ocean_props_atlas_v001.png`: 64×64 셀, 4×2 grid, 8종 장식
- 생성 원본과 chroma-key 원본은 `art_source/concepts/`, 투명화 중간본은 `art_source/processed/`에 보존
- `tool/build_environment_assets.py`가 runtime 타일과 아틀라스를 nearest-neighbor 방식으로 재생성
- 장식은 배경 레이어 전용이며 이동·전투·스폰 충돌에는 참여하지 않음
- 430×932 Web 검수에서 타일 경계, 링 대비, UI 가독성과 장식 밀도를 확인

### M8 픽셀 UI 체크포인트

- 공통 팔레트·이중 프레임·3px 그림자·각진 pressed 상태를 코드 기반 UI 토큰으로 관리
- HP·Fullness·EXP는 10칸 분절 게이지와 중앙 수치를 함께 표시
- 도감·종변화 초상화는 종별 swim strip 첫 프레임을 nearest-neighbor로 크롭
- 잠금·발견·해금·현재·선택 상태는 색상과 기호를 동시에 사용
- NPC 관계는 초록 `+`, 노랑 `=`, 빨강 `!`로 색각 외 보조 단서 제공
- 한글 본문은 가독성을 위해 시스템 글꼴을 유지하고 영문 제목·숫자·프레임에 픽셀 감성을 집중
- 430×932와 390×844 Web 검수에서 카드, 가로 종 선택 목록과 하단 버튼 오버플로 없음 확인

### M8 전투 피드백 체크포인트

- bite는 흰 이빨과 4방향 파편, 피격은 8방향 적색 파편, 포식은 초록 확산으로 구분
- 레벨업은 금색 상승 파티클, 종 해금은 더 길고 큰 보라색 상승 파티클 사용
- `assets/audio/`의 WAV 4종은 22.05kHz mono 16-bit PCM이며 `tool/build_audio_assets.dart`로 재생성
- 외부 음원 없이 파형을 직접 합성해 라이선스 출처와 빌드 재현성 문제를 제거
- 효과음·햅틱 실패는 시각 피드백과 전투 판정에 영향을 주지 않도록 분리

## 10. 에셋 완료 기준

- 100% 확대에서 픽셀 경계가 깨끗함
- 게임 내 배율에서 종을 즉시 구분 가능
- 좌우 반전 시 빛 방향과 비대칭 요소가 어색하지 않음
- atlas 경계에서 다른 스프라이트 픽셀이 새지 않음
- 웹과 모바일 모두 블러 없이 표시
- 애니메이션 루프가 튀지 않음
- 위험도 UI가 배경색과 무관하게 식별 가능
- 파일명, manifest, 라이선스/출처 기록이 존재
