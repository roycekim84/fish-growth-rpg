# Legacy: 모바일 2D 물고기 성장/사냥 게임 — 1차 프로토타입 기획·구현 명세

> 현재 문서는 초기 성장/사냥 프로토타입의 구현 기준이다.
> 2026-07-20 이후 제품 방향은 `docs/MASTER_SPEC.md`의 탐험 어드벤처 RPG 기준을 우선한다.
> 이 문서의 시스템은 폐기 대상이 아니라 탐험을 보조하는 하위 시스템으로 재분류한다.

## 1. 문서 목적

이 문서는 모바일 세로 화면 기반의 2D 물고기 성장/사냥 게임 1차 프로토타입을 구현하기 위한 기준 문서다.

프로토타입의 핵심 검증 대상은 다음 네 가지다.

1. 작은 물고기를 사냥하고 성장하는 재미
2. 큰 물고기를 피해야 하는 긴장감
3. 먹이를 먹고 안전한 곳에서 쉬며 회복하는 흐름
4. 같은 종을 100마리 포식해 새로운 종을 해금하는 장기 목표

그래픽 완성도보다 핵심 게임 루프와 확장 가능한 데이터 구조를 우선한다.

확정된 제품·기술 방향:

- Flutter stable + Flame 기반 단일 코드베이스
- Android Google Play 및 iOS App Store 출시 목표
- 개발 중 Flutter Web으로 빠른 반복 테스트 지원
- 모바일 세로 화면 전용 플레이
- 배경, 캐릭터, 이펙트, UI를 포함한 전체 픽셀아트 스타일
- Git 커밋과 Markdown 문서를 기준으로 단계별 진행

## 3. 핵심 게임 루프

```text
탐색 및 이동
  → 먹을 수 있는 NPC 발견
  → 추격 및 충돌
  → 포식 또는 전투
  → EXP·배부름·종별 포식 횟수 획득
  → 레벨업 또는 종변화 해금
  → 위험 회피
  → 안전한 곳에서 정지해 HP 회복
  → 다시 탐색
```

플레이어가 항상 판단해야 하는 핵심 질문은 다음과 같다.

- 이 물고기를 바로 먹을 수 있는가?
- 싸우면 이길 수 있는가?
- 지금은 도망쳐야 하는가?
- 배부름을 더 채울 것인가, 안전한 곳에서 회복할 것인가?
- 다음 종변화까지 몇 마리가 남았는가?

## 4. 플레이 상태와 화면 구성

### 4.1 기본 플레이 화면

- 상단: HP, Fullness, EXP, 레벨, 현재 종
- 중앙: 플레이어를 따라가는 2D 게임 필드
- 대상 물고기: 위험도에 따른 외곽선 표시
- 하단 또는 접이식 영역: 종별 포식 진행도 요약
- 오른쪽 하단: 대시/부스터 버튼 1개
- 메뉴 버튼: 도감, 종변화, 설정/저장 진입

### 4.2 주요 화면 상태

```text
Boot
  → Load/Initialize Save
  → Playing
      ↔ Codex UI
      ↔ Species Change UI
      → Player Defeated
```

- 도감과 종변화 UI가 열려 있는 동안 게임 월드는 일시 정지하는 것을 기본 정책으로 한다.
- 플레이어 사망 시 즉시 저장 데이터를 삭제하지 않는다.
- 사망 처리 방식은 구현 전 확정이 필요하며, 초기 권장안은 현재 종과 진행도를 유지한 채 HP를 완전히 회복하고 안전 위치에서 재시작하는 방식이다.

## 5. 조작과 이동

### 5.1 입력

- 게임 화면을 드래그하면 드래그 방향을 목표 이동 방향으로 사용한다.
- 입력 영역은 UI 버튼과 패널을 제외한 전체 게임 화면이다.
- 손가락을 떼면 추진 입력을 0으로 만들고 관성에 따라 서서히 감속한다.
- 오른쪽 하단 부스터 버튼을 누르는 동안 이동 속도 또는 추진력을 일시적으로 높인다.
- 별도의 공격 버튼은 없다.

### 5.2 권장 이동 모델

```text
desiredDirection = normalize(currentPointer - dragStartOrAnchor)
velocity += desiredDirection * acceleration * deltaTime
velocity = clampMagnitude(velocity, currentMaxSpeed)
velocity *= dragFactor
position += velocity * deltaTime
```

- 물고기의 시각적 방향은 현재 이동 방향을 향해 부드럽게 회전한다.
- 좌우 방향 전환 시 스프라이트 또는 도형을 수평 반전한다.
- 필드 경계 밖으로 나가지 않도록 위치를 제한하고, 경계 충돌 시 속도의 바깥 방향 성분을 제거한다.
- DEX는 데이터에는 포함하되, 1차 구현에서는 회전 보간 속도에만 약하게 반영하거나 아직 사용하지 않아도 된다.

### 5.3 부스터의 최소 규칙

- 부스터 사용 중 회복 불가
- 부스터 사용 중 최대 속도 증가
- 초기 버전에서는 별도의 마나 게이지를 추가하지 않는다.
- 무한 사용이 플레이를 해치면 짧은 쿨다운 방식으로 조정한다.

### 5.4 반자동 사냥

반자동 사냥은 방치형 진행이 아니라 반복 조작을 줄이는 선택형 보조 기능으로 구현한다.

- 탐지 범위 안에서 즉시 포식 가능한 가장 가까운 초록 대상을 자동 선택
- 자동 이동 속도는 수동 기본 속도의 75%
- 자동 부스터 사용 없음
- 150 logical pixels 안에 빨간 위험 대상 접근, HP 35% 이하, 수동 드래그 입력 시 즉시 중단
- 위험 또는 저체력 중단 시 자동 추진으로 남은 관성을 제거
- 포식 판정과 보상은 수동 플레이와 동일
- 회복, 종변화, 도감과 성장 선택은 플레이어가 직접 수행
- 앱이 백그라운드이거나 종료된 동안에는 동작하지 않음

자동사냥이 수동 플레이보다 안전하거나 효율적이어서는 안 된다. M4.5 기준 0.15초마다 대상을 다시 탐색하며, 경로 계획이나 위험 우회는 제공하지 않는다. 후속 웹 알파 플레이 데이터에 따라 75% 속도와 150 거리만 조정한다.

## 6. 물고기 종 데이터

모든 종은 코드 분기보다 데이터 정의를 통해 추가할 수 있어야 한다.

### 6.1 공통 스키마

```text
FishSpecies
- id: String
- displayName: String
- description: String
- behaviorType: BehaviorType
- baseMaxHP: Float
- baseSTR: Float
- baseDEX: Float
- baseINT: Float
- baseSPD: Float
- baseSize: Float
- baseWeight: Float
- expReward: Int
- fullnessReward: Float
- unlockEatCount: Int = 100
- playerStatMultipliers: StatMultipliers
- visual: FishVisualDefinition
- traits: List<TraitId>
- spawnConfig: SpawnConfig
```

`base*` 값은 NPC 개체의 기본 스탯과 종변화 시 플레이어 배율 계산의 원천 데이터로 활용할 수 있다. 플레이어의 영구 성장 스탯과 현재 종 배율은 서로 분리한다.

### 6.2 초기 종 정의

| ID | 표시명 | 역할 | HP | STR | SPD | Size | Weight | EXP | Fullness | 최대 개체 수 |
|---|---|---|---:|---:|---:|---:|---:|---:|---:|---:|
| `small_fish` | 작은 물고기 | 빠르고 약한 초반 먹이 | 10 | 1 | 2.2 | 0.6 | 0.5 | 5 | 10 | 30 |
| `puffer_fish` | 복어 | 느리고 튼튼한 중간 위험 | 35 | 4 | 1.1 | 1.0 | 1.5 | 20 | 25 | 10 |

DEX와 INT의 초기값, 플레이어 시작 종, 각 종의 플레이어용 스탯 배율은 밸런스 설정 단계에서 확정한다.

## 7. 플레이어 스탯 모델

### 7.1 영구 성장 스탯과 종 배율 분리

종변화 후에도 레벨 성장을 보존하기 위해 최종 스탯을 다음처럼 계산한다.

```text
progressionStat = basePlayerStat + levelGrowth
finalStat = progressionStat * currentSpeciesMultiplier
```

예시:

```text
finalMaxHP = (baseMaxHP + (level - 1) * 5) * species.maxHPMultiplier
finalSTR   = (baseSTR   + (level - 1) * 1) * species.strMultiplier
finalSize  = (baseSize  + (level - 1) * 0.03) * species.sizeMultiplier
```

이 구조를 사용하면 종 추가 시 레벨 시스템을 수정하지 않고 배율과 특성만 추가할 수 있다.

### 7.2 레벨업

```text
RequiredExp(level) = 20 + level * 10
```

레벨업 증가량:

- MaxHP +5
- STR +1
- Size +0.03
- Weight +0.05
- SPD는 초기 제안에 증가량이 없으므로 기본적으로 증가시키지 않는다.

한 번의 보상으로 여러 레벨에 필요한 EXP를 얻을 수 있으므로 `while exp >= requiredExp` 형태로 처리한다.

레벨업 시 권장 정책:

- 현재 HP는 증가한 MaxHP만큼 같이 올려 주되 MaxHP를 넘지 않는다.
- EXP는 필요량만 차감하고 나머지를 다음 레벨로 이월한다.

## 8. 포식과 전투

### 8.1 충돌 판정

플레이어와 살아 있는 NPC가 충돌했을 때, 플레이어 관점에서 다음 순서로 판정한다.

| 조건 | 결과 | 표시 |
|---|---|---|
| `playerSize >= npcSize * 1.15` | NPC 즉시 포식 | 초록 |
| `playerSize < npcSize * 0.9` | 플레이어가 피해를 받음 | 빨강 |
| 그 외 | 서로 물기 피해 교환 | 노랑 |

보라색은 일반 위험도와 별개의 특수 표식이다. 종변화 목표 강조에 사용할 경우 위험도 색상을 가리지 않도록 아이콘 또는 보조 링으로 표현한다.

### 8.2 접촉 피해 제어

물리 프레임마다 피해가 반복되지 않도록 개체 쌍별 공격 쿨다운이 필요하다.

초기 권장값:

- 물기 판정 간격: 0.75초
- 전투 후 회복 금지 시간: 마지막으로 피해를 주거나 받은 시점부터 1초
- 피해량: 공격자의 STR
- 최소 피해: 밸런스상 필요하면 1 보장

`Weight`, `SPD`, `DEX`가 포함된 복합 CombatPower는 1차 프로토타입의 실제 판정에서는 사용하지 않는다. 추후 AI 판단 또는 전투 확장용으로 남겨 둔다.

플레이어 성장 기본값은 MaxHP 40, STR 3, Size 0.8, Weight 1.0이다. 플레이어 사망 시 월드 중앙에서 HP를 전부 회복하고 1.5초 동안 접촉 피해를 받지 않는다. M5부터 이 기본값에 영구 레벨 성장량을 더하며, M6에서 종 배율을 추가한다.

### 8.3 사망과 포식 보상

NPC HP가 0 이하가 되면 포식 성공으로 처리하며 즉시 포식과 같은 보상을 지급한다.

- EXP 증가
- Fullness 증가, MaxFullness에서 제한
- 해당 NPC 종의 포식 카운트 +1
- 포식 카운트가 해금 기준 이상이면 해당 종 해금
- 도감 발견/포식 기록 갱신
- NPC 제거 후 재생성 예약

보상으로 HP를 직접 회복하지 않는다.

### 8.4 중복 처리 방지

- NPC는 `Alive → Dying/Consumed → Despawned` 상태를 가진다.
- `Consumed` 전환은 한 번만 가능해야 한다.
- 보상 지급, 카운트 증가, 재생성 예약 역시 한 번만 실행한다.

## 9. HP, 배부름, 정지 회복

### 9.1 상태값

- `hp`, `maxHP`
- `fullness`, `maxFullness`
- `idleDuration`
- `lastCombatTime`
- `isBoosting`
- `isRecovering`

M5 초기값은 Fullness 50 / MaxFullness 100이다. 포식 보상이 상한을 넘을 때 HUD와 메시지에는 실제 증가량만 표시한다.

### 9.2 회복 진입 조건

모든 조건을 만족할 때만 회복한다.

```text
movementInputMagnitude <= idleInputThreshold
AND velocityMagnitude <= idleVelocityThreshold
AND idleDuration >= 1.5 seconds
AND timeSinceLastCombat >= 1.0 second
AND isBoosting == false
AND fullness > 0
AND hp < maxHP
```

입력만 놓았지만 관성으로 빠르게 이동 중인 상황은 정지로 보지 않는다.

### 9.3 회복 처리

- 초당 Fullness 5 소모
- 초당 HP 8 회복
- 프레임 독립적으로 `rate * deltaTime` 적용
- Fullness가 0이 되거나 움직이면 즉시 중단
- HP가 MaxHP가 되면 중단
- 한 프레임에서 남은 Fullness보다 많은 회복 비용을 쓰지 않도록 실제 소모량에 비례해 HP를 회복

M5 구현에서는 입력 또는 관성 속도가 임계값을 넘으면 idle 시간을 0으로 되돌린다. 회복 중에는 플레이어 외곽에 초록 픽셀 펄스를 표시하고 HUD 상단에 `RECOVER`를 표시한다.

## 10. NPC AI

### 10.1 상태

```text
Wander
Flee
Chase
```

### 10.2 전환 규칙

- 플레이어가 감지 범위 안에 있고 플레이어가 자신을 즉시 포식할 수 있으면 `Flee`
- HunterFish는 플레이어가 자신의 Size 0.9배 미만이면 `Chase`
- PufferFish는 방어적 성격을 유지하고 플레이어가 자신의 Size 0.7배 미만일 때만 `Chase`
- SmallFish는 플레이어가 자신의 Size 0.72배 미만일 때만 `Chase`
- 그 외에는 `Wander`
- 크기가 비슷한 경우 종의 `behaviorType`에 따라 행동
  - SmallFish: Flee 우선
  - PufferFish: Wander 유지
  - HunterFish: Chase 우선

권장 감지 거리: 5 world units. 거리 수치는 실제 화면 크기와 월드 스케일을 확정한 뒤 조정한다.

### 10.3 상태별 행동

- Wander: 일정 간격마다 랜덤 목표 방향 또는 목표 위치 선택
- Flee: 플레이어 반대 방향으로 이동
- Chase: 플레이어 위치 방향으로 이동
- 경계 근처에서는 필드 중심 방향 성분을 추가
- 방향이 매 프레임 급변하지 않도록 목표 방향을 보간

AI 업데이트는 모든 렌더 프레임마다 수행할 필요가 없다. 상태 판단은 낮은 주기, 이동 적용은 매 프레임으로 분리할 수 있다.

M3 기준 상태 판단 주기는 플레이어와 가까울 때 0.15초, 감지 거리의 2.5배보다 멀 때 0.6초다. 이동 보간은 매 프레임 수행한다.

## 11. 스폰과 필드

### 11.1 필드

- 단순한 사각형 월드
- 카메라는 플레이어를 부드럽게 추적
- 플레이어와 NPC 모두 경계 밖 이동 불가
- 세로 화면에서 카메라 밖에도 탐색 공간이 있도록 월드를 화면보다 크게 구성

### 11.2 스폰 규칙

- 종마다 최대 활성 개체 수 유지
- 플레이어의 최소 스폰 거리 밖에서 생성
- 필드 경계 안의 무작위 위치 사용
- 카메라 안에서 갑자기 나타나는 현상을 줄이기 위해 가능하면 화면 밖 위치 우선
- 포식된 NPC는 짧은 지연 후 재생성
- 유효한 위치를 못 찾으면 다음 스폰 주기에 재시도

초기 권장값:

- 최소 플레이어 거리: 화면 짧은 축의 약 40% 또는 이에 대응하는 월드 거리
- 재생성 지연: 2~4초 무작위
- 한 프레임에 대량 생성하지 않도록 생성 수 제한

## 12. 종변화 시스템

### 12.1 해금

```text
eatenCountBySpeciesId[speciesId] >= species.unlockEatCount
```

- 기준값은 종 데이터에 저장하며 초기값은 100이다.
- 이미 해금된 종은 다시 해금 이벤트를 발생시키지 않는다.
- 해금 시 명확한 화면 알림을 표시한다.

### 12.2 종변화 시 유지

- 레벨
- 현재 및 누적 경험치 정책
- 해금 종 목록
- 종별 포식 카운트
- 도감 기록
- 마지막 저장 시간

### 12.3 종변화 시 변경

- 현재 종 ID
- 외형
- 최종 MaxHP, STR, SPD, Size, Weight 배율
- 종 특성

### 12.4 종변화 실행 조건

- 선택 종이 해금되어 있어야 함
- 현재 전투 중이 아니어야 함
- 종변화 비용 없음
- UI가 게임을 일시 정지하므로 회복 중 종변화는 허용 가능

종변화 후 HP 처리 권장안:

```text
hpRatio = oldHP / oldMaxHP
newHP = clamp(newMaxHP * hpRatio, 1, newMaxHP)
```

체력 비율을 유지하면 종변화가 무료 즉시 회복 수단이 되는 것을 방지할 수 있다.

M6 구현 기준선:

- 시작 종 ID는 `starter_fish`이며 항상 해금한다.
- SmallFish 배율: MaxHP 0.75, STR 0.75, SPD 1.20, Size 0.80, Weight 0.70
- PufferFish 배율: MaxHP 1.35, STR 1.00, SPD 0.78, Size 1.15, Weight 1.40
- HunterFish 배율: MaxHP 1.10, STR 1.35, SPD 1.10, Size 1.25, Weight 1.25
- 전투 직후 1초 동안 종변화를 제한한다.
- 도감과 종변화 오버레이를 표시하는 동안 게임 월드를 일시 정지한다.

### 12.5 종 특성 확장 구조

초기에는 배율 차이만 구현해도 되지만, 다음 인터페이스를 고려한다.

```text
FishTrait
- onEquipped(player)
- onUnequipped(player)
- onDealDamage(context)
- onTakeDamage(context)
- onConsumeFish(context)
- update(deltaTime)
```

복어 반사 피해 같은 능력은 이 구조에 후속 추가한다.

## 13. 도감과 종변화 UI

### 13.1 도감 항목

- 종 이름과 임시 외형
- 발견 여부
- 포식 수 `현재 / 해금 기준`
- 해금 여부
- 해금까지 남은 수
- 기본 스탯
- 역할 및 특성 설명

미발견 종을 완전히 숨길지 실루엣으로 보여 줄지는 구현 전 결정한다. 프로토타입에서는 목표 전달을 위해 잠긴 실루엣과 `0 / 100`을 표시하는 편이 적합하다.

### 13.2 종변화 화면

- 전체 초기 종 목록 표시
- 해금 종은 선택 가능
- 미해금 종은 잠금 아이콘과 진행도 표시
- 현재 종 강조
- 선택 후보의 스탯 변화 미리 보기
- 확인 시 종변화 실행
- 전투 중 열려고 하면 불가 사유 안내

## 14. 로컬 저장

### 14.1 저장 데이터

```text
PlayerSaveData
- schemaVersion: Int
- level: Int
- exp: Int
- currentSpeciesId: String
- hp: Float
- fullness: Float
- unlockedSpeciesIds: Set<String>
- eatenCountBySpeciesId: Map<String, Int>
- discoveredSpeciesIds: Set<String>
- lastSaveTimeUtc: ISO-8601 String
```

### 14.2 저장 정책

- 앱 시작 시 로드, 데이터가 없으면 기본값 생성
- 포식, 레벨업, 종 해금, 종변화처럼 중요한 이벤트 후 저장 요청
- 지나친 디스크 기록을 피하도록 짧은 디바운스 적용
- 앱이 백그라운드로 가거나 종료 이벤트를 받을 때 저장
- 시간은 UTC로 저장
- `schemaVersion`으로 향후 마이그레이션 지원
- 알 수 없는 종 ID나 누락 필드는 기본값으로 복구

M7 구현 기준선:

- 저장 키: `fish_growth_player_save_v1`
- 저장 매체: `SharedPreferencesAsync`를 감싼 `PlayerSaveRepository`
- 진행 상태 변경 후 700ms 디바운스 저장
- 종변화와 앱 inactive/paused/detached 시 즉시 저장
- 손상 JSON은 제거하고 기본 진행도로 복구
- 현재 앱보다 높은 schema는 삭제하거나 덮어쓰지 않음
- HUD에 현재 저장 상태를 짧은 영문 라벨로 표시

월드 내 NPC 위치와 HP는 저장하지 않는다. 세션 진행도와 영구 진행도만 저장한다.

Flutter는 앱 수명주기, 메뉴, HUD와 오버레이를 담당하고 Flame은 실시간 게임 루프, 월드, 카메라, 입력, 충돌과 렌더링을 담당한다.

핵심 원칙:

- UI는 시스템 상태를 읽고 명령을 전달하되 규칙 자체를 소유하지 않는다.

- 웹은 개발·QA 플랫폼으로 사용하고 모바일 터치 조작과 성능을 출시 기준으로 삼는다.

## 17. 검증 시나리오

### 이동

- 드래그 방향으로 회전하고 이동한다.
- 손을 떼면 즉시 멈추지 않고 감속한다.
- 부스터 사용 중 속도가 증가하고 회복되지 않는다.
- 모든 개체가 필드 밖으로 나가지 않는다.

### 포식과 전투

- 크기 1.15배 이상인 대상은 즉시 포식한다.
- 크기 0.9배 미만인 상대와 접촉하면 플레이어가 피해를 받는다.
- 중간 구간에서는 쿨다운에 따라 서로 피해를 준다.
- 같은 NPC에서 보상이 두 번 지급되지 않는다.

### 회복

- 1.5초 이상 실질적으로 정지하면 회복한다.
- 초당 Fullness 5, HP 8 비율로 처리된다.
- 이동, 부스터, Fullness 0, MaxHP에서 회복이 멈춘다.
- 마지막 전투 후 1초 동안 회복하지 않는다.

### 성장

- EXP가 요구량에 도달하면 레벨업한다.
- 초과 EXP가 이월된다.
- 여러 레벨 분량의 EXP도 정상 처리된다.
- 레벨업 후 최종 스탯과 UI가 즉시 갱신된다.

### 종변화

- 99마리에서는 잠겨 있고 100마리에서 정확히 해금된다.
- 해금 알림은 한 번만 발생한다.
- 전투 중에는 종변화할 수 없다.
- 종변화 후 레벨, EXP, 카운트, 도감 기록이 유지된다.
- 외형과 종 배율이 적용되고 HP 비율이 유지된다.

### 저장

- 앱 재실행 후 레벨, EXP, 종, 해금, 카운트가 복원된다.
- 마지막 저장 시간이 UTC로 기록된다.
- 저장 데이터 일부가 누락되어도 기본값으로 복구된다.

## 18. 프로토타입 성공 기준

다음 조건을 모두 충족하면 1차 프로토타입의 핵심 목표를 달성한 것으로 본다.

- 모바일에서 드래그와 버튼 하나만으로 조작할 수 있다.
- 먹을 수 있는 대상, 싸울 대상, 피할 대상을 즉시 구분할 수 있다.
- 작은 물고기를 추격하고 먹는 행동이 반복해도 답답하지 않다.
- 강한 물고기가 실제 위협으로 기능한다.
- HP가 낮을 때 먹이를 확보하고 정지 회복하는 선택이 의미가 있다.
- 레벨업으로 성장하는 감각이 있다.
- 종별 `100마리` 목표와 현재 진행도가 명확하다.
- 최소 2개 이상의 종변화 선택에서 스탯 차이가 체감된다.
- 앱을 재실행해도 주요 진행도가 보존된다.

## 19. 구현 전 확정이 필요한 결정

아래 항목은 원 요구사항만으로 단일 답이 정해지지 않으므로 실제 구현 착수 전에 결정한다.

1. 플레이어의 최초 종과 최초 해금 종 목록
2. 플레이어 기본 스탯 및 종별 플레이어 스탯 배율
3. DEX와 INT의 초기값 및 1차 버전 적용 여부
4. 플레이어 사망 이후 진행도 손실 또는 추가 페널티
5. 부스터 배율, 지속 방식, 쿨다운
6. 비슷한 크기의 전투에서 NPC가 플레이어에게 받는 정확한 피해량과 반격 방식
7. 현재 EXP와 누적 EXP를 별도로 저장할지 여부
8. 도감의 미발견 종 노출 방식
9. 월드 크기, 월드 단위와 화면 픽셀의 대응 관계

### 권장 기본 결정

- 플레이어 시작 종: `small_fish` 계열의 기본 종
- 최초 시작 종은 자동 해금
- 일시 정지형 도감/종변화 UI
- HP 비율 유지형 종변화
- 안전 위치 부활, 진행도 유지
- M4 기준 월드 중앙 부활, HP 완전 회복, 1.5초 접촉 무적
- 데이터 기반 종 정의
- 부스터는 1.5배 속도와 짧은 쿨다운으로 시작 후 플레이 테스트로 조정

## 20. 후속 확장 방향

1차 구조를 유지하면서 다음 기능을 추가할 수 있어야 한다.

- 종별 패시브 및 액티브 능력
- 추가 AI 상태와 무리 행동
- 지역별 스폰 테이블
- 장비와 성장 트리
- 최대 8시간 오프라인 보상
- 보스와 특수 개체
- 계정 및 서버 저장
- 랭킹과 멀티플레이

온라인 기능은 로컬 핵심 루프가 검증된 뒤 별도 아키텍처 단계에서 다룬다.
